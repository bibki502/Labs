Тема: Практическое применение контейнеров и итераторов в C++

1. Цель работы
Изучение механизмов работы со стандартной библиотекой шаблонов (STL), освоение различных типов контейнеров (std::vector, std::list, std::deque) и методов навигации по ним с помощью итераторов.

2. Описание выполненных задач
Задача 1. Оптимизация списка товаров
Суть: Работа с std::vector<int>. Программа принимает цены, сортирует их через std::sort с компаратором std::greater. Среднее арифметическое вычисляется через проход итератором, после чего элементы ниже среднего удаляются методом erase.

Используемые инструменты: std::vector, std::sort, std::accumulate.

Задача 2. Управление инвентарём
Суть: Демонстрация работы std::back_inserter. Пользователь вводит названия предметов, которые добавляются в список. Итераторы-адаптеры позволяют безопасно добавлять данные в конец контейнера без ручного управления индексами.

Используемые инструменты: std::list<std::string>, std::back_inserter.

Задача 3. Редактор журнала
Суть: Чтение файла journal.txt построчно в std::list<std::string>. Итераторы позволяют найти нужную строку, изменить её содержимое и сохранить обновленный список обратно в файл.

Используемые инструменты: std::fstream, итераторы для поиска и замены строк.

Задача 4. Синхронизация данных
Суть: Копирование данных из std::vector в std::deque и обратно. Задача демонстрирует совместимость итераторов: алгоритм std::copy работает одинаково успешно с разными типами контейнеров, так как оба поддерживают последовательный доступ.

Используемые инструменты: std::vector, std::deque, std::copy.

Задача 5. Реверсивное отображение данных
Суть: Использование reverse_iterator (методы rbegin() и rend()). Программа считывает список строк и выводит их в обратном порядке без изменения самого контейнера в памяти.

Используемые инструменты: std::vector::rbegin, std::vector::rend.

3. Результаты выполнения
В ходе работы была настроена среда Docker для сборки и тестирования многофайловых проектов. Код успешно скомпилирован с использованием g++. Все операции над контейнерами (вставка, удаление, поиск и реверс) выполнены с использованием итераторов, что обеспечивает универсальность и безопасность кода.

4. Вывод
Лабораторная работа позволила на практике понять абстракцию итераторов. Главный вывод: итераторы позволяют писать алгоритмы (сортировка, копирование, поиск), которые не зависят от внутренней структуры контейнера, будь то массив в памяти или связный список.
