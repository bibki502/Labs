---

### 2. Описание выполненных задач

#### Задача 1: Оптимизация списка товаров
**Задание:** Запросить список цен, отсортировать по убыванию и удалить те, что ниже среднего значения.
**Реализация:** * Использован *std::vector<int>* для хранения цен.
* Сортировка выполнена через *std::sort* с итераторами *begin()* и *end()*.
* Среднее значение вычислено с помощью *std::accumulate*.
* Удаление элементов произведено методом `erase` в связке с алгоритмом *std::remove_if*.

#### Задача 2: Управление инвентарём
**Задание:** Реализовать добавление и удаление элементов через командную строку.
**Реализация:**
* Использован *std::list<std::string>*, так как он эффективен при частом добавлении/удалении.
* Применен адаптер итератора *std::back_inserter* для добавления элементов в конец списка.

#### Задача 3: Редактор журнала
**Задание:** Загрузка текста из файла в список строк, редактирование и сохранение обратно.
**Реализация:**
* Использован *std::fstream* для работы с файловой системой.
* Контент хранится в *std::list<std::string>*, где каждая строка — отдельный элемент.
* Поиск нужной строки для редактирования реализован через проход обычным итератором.

#### Задача 4: Синхронизация данных
**Задание:** Скопировать данные из вектора в двухстороннюю очередь (deque) и обратно.
**Реализация:**
* Демонстрация работы алгоритма *std::copy*.
* Итераторы позволили абстрагироваться от типа контейнера: один и тот же алгоритм успешно перенес данные из *vector::begin()* в *deque::begin()*.

#### Задача 5: Реверсивное отображение данных
**Задание:** Считать список имен/дат и вывести их в обратном порядке.
**Реализация:**
* Использованы обратные итераторы *rbegin()* и *rend()*.
* Это позволило вывести данные задом наперед, не меняя их порядок в самой памяти.
