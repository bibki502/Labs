### 2. Описание реализованных итераторов

#### 1. Пользовательский обратный итератор (`ReverseIterator.h`)
* **Назначение:** Обход истории изменений (версий документа) от новых к старым.
* **Логика:** Инвертирует направление движения: оператор `++` внутреннего итератора заменен на `--`.
* **Особенность:** Реализован сдвиг при разыменовании для корректной работы с границами контейнера (аналогично `std::reverse_iterator`).

#### 2. Итератор временного интервала (`IntervalIterator.h`)
* **Назначение:** Фильтрация событий в журнале по заданному диапазону (например, времени или датам).
* **Логика:** Внутри итератора инкапсулирована функция `skipInvalid()`, которая при каждом шаге пропускает элементы, выходящие за границы `[minV, maxV]`.

#### 3. Итератор фильтрации по критерию (`FilterIterator.h`)
* **Назначение:** Выделение задач из общего списка по произвольному условию (например, только высокий приоритет).
* **Логика:** Использует `std::function` в качестве предиката. Позволяет передавать лямбда-выражения для гибкой настройки фильтра прямо в коде.

#### 4. Итератор трансформации данных (`TransformIterator.h`)
* **Назначение:** Изменение данных "на лету" без создания копии контейнера (конвертация валют, форматирование).
* **Логика:** При разыменовании (`operator*`) возвращает не сам элемент, а результат применения функции трансформации к нему.

#### 5. Мульти-итератор (`MultiIterator.h`)
* **Назначение:** Бесшовное объединение разнородных источников (например, список email и список телефонов).
* **Логика:** Хранит границы двух контейнеров. При достижении конца первого контейнера автоматически переключает внутренний указатель на начало второго.

---

### 3. Технические особенности реализации

* **Шаблоны (Templates):** Все итераторы реализованы как шаблоны классов, что позволяет им работать с любыми типами данных и контейнерами.
* **Совместимость с STL:** В каждом классе определены необходимые псевдонимы типов (`value_type`, `iterator_category`, `difference_type`), что позволяет использовать их в стандартных алгоритмах.
* **Header-only:** Код вынесен в заголовочные файлы (`.h`), так как реализация шаблонов должна быть доступна при компиляции `main.cpp`.
